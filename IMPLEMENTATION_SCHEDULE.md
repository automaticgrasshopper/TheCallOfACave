## 1. 结论

现有主循环可以作为正式底座，叙事流程也已经足以确定游戏的完整方向。把本文件中的默认规则纳入需求后，代码、美术、UI、演出、无配音对白、背景音乐和基础音效可以组成一个完整、可通关、可存档并完成最终反转的 1.0 版本。

计划完成的完整体验是：

`创建档案 → 第一纪元引导与繁衍建设 → 第二纪元城市与防守 → 第三纪元集结进攻 → 胜利假结局 → 虫族驾驶机器人反转 → 交棒给受赠人 → Logo 与制作人员字幕`

目前真正的外部依赖只有两项：

1. Unity 必须有可用的 2022.3.44f1 编辑器和许可证，才能做最终自动构建验证。
2. 音乐若使用第三方生成或素材服务，必须取得可用于商业游戏的授权并保存授权凭证；若届时没有外部音乐来源，则使用本地制作的原创合成音乐，不阻塞功能完成，但成品质感可能低于专业作曲版本。

## 2. 1.0 范围锁定

- 平台：PC，键盘与鼠标；主发布目标默认为 Windows x64，macOS 用于开发验证。
- 基准画面：640×360 像素画布，按现有规范整数倍显示；正式 UI 以 1920×1080 基准适配。
- 语言：1.0 随附简体中文和英文；存档姓名支持任意 Unicode 语言。繁体中文及其他语言保留接口，但不计入本工期。
- 无角色配音。德尔塔只使用文字、立绘、表情切换和轻量提示音。
- 音乐：主菜单、第一纪元、第二纪元、第三纪元、结尾/反转，共 5 组可循环主题。
- 音效：明确点击按钮、确认/取消、拖放成功/失败、出售、建造、升级、纪元变化、宝箱落地/开启、反应堆爆炸、胜利等。普通虫群交战保持静默。
- Hover 只用于明确可点击按钮；可拖动的虫、蛋、物品、建筑卡不套按钮 Hover，只保留拖起、合法落点、非法落点和放下反馈。
- 1.0 不包含：联网、云存档、Steam 成就、手柄、配音、创意工坊、关卡编辑器和移动端。

## 3. 已补齐的设计坑与默认规则

这些规则是为了保证后续可以直接实施。若未来要改，必须在对应开发日开始前修改本节，否则按本节执行。

### 3.1 档案、命名与重开

- 每个档案保存两个永久字段：玩家姓名、研究成果受赠人姓名。
- 两个姓名均接受任意 Unicode；去除首尾空白后为 1–24 个用户可见字符；不允许纯空白。
- 玩家姓名一经确认不可修改。改名只能删除该档案后重建；受赠人姓名同样锁定，保证结尾反转不会失去含义。
- 档案列表按最后游玩时间倒序，默认选中最近使用的档案；支持无限滚动，不预设固定 Slot 数量。
- Slot 显示玩家姓名、累计游玩时长、当前纪元；通关后阶段显示“新时代”。
- “开始游戏”始终创建一个新档案；“继续游戏”直接进入当前选中的档案；“档案管理”负责切换当前档案。
- “继续游戏”永远继续当前选中的档案。没有档案时，该按钮进入创建档案流程。
- 新档案第一次进入引导时资金为 200；强制出售一个价值 100 的蛋后变为 300。
- 同一档案此后的重新开始资金为 300，并跳过完整新手引导；继续游戏则恢复保存时的真实资金和世界状态。
- Game Over 不删除档案。玩家可读取最近自动保存，或重开本轮；重开保留两个姓名、累计时长和“已看过引导”标记。
- 自动保存每 60 秒一次，并在建造、升级、纪元变化、进入/退出、胜利等关键节点保存。
- 存档采用带版本号的数据结构，先写临时文件再替换正式文件；损坏时保留原文件并回退到上一个有效快照。

### 3.2 开场与新手引导

- 点击开始后先完成两次命名，再通过现有 Block 进入主场景。
- 主场景初始暂停、右侧功能 UI 隐藏、场上无虫。德尔塔第一次提到“幼体和蛋”时，样本才进入场地，避免对白与画面矛盾。
- 引导样本固定为 3 只幼体、场地 1 枚蛋、背包 1 枚教学蛋；教学蛋只能拖到金币区域，其他操作暂时锁定。
- 第一次引导不可跳过，避免玩家绕过 200→300 的资金教学。看过后，重开本轮不再播放完整引导。
- 德尔塔统一从右侧推入/退出；对白框位于下方偏左。立绘切换使用短暂交叉渐变，不瞬切。
- 对白逐句点击推进，句尾右下角角标循环跳动；对话期间模拟暂停，只允许当前教学要求的输入。
- 首只幼体长成成虫时只触发一次第二段教学，并显示建筑栏目。

### 3.3 第一纪元与辐射宝箱

- 年份前始终显示当前纪元；进入每一纪元时显示中央大型 Toast，年份连续累加，不重置。
- 第一纪元完成条件：本轮时间至少 12 分钟，并且工厂、兵营、医院、研究院各至少有一座达到 Lv.3。重复建造的同类建筑不要求全部满级。
- 条件未满足时，敌人继续按已有节奏袭扰；满足后只触发一次德尔塔过场。
- 辐射宝箱第一次在 150 秒出现，之后随机间隔 90–150 秒；落点先显示 1.2 秒预警，再播放坠落。
- 最近的一只空闲成虫主动接近并开启宝箱；没有合格成虫时，宝箱留在原地等待。
- 默认掉落权重：食物 35%、零件 30%、精品装备 20%、辐射爆炸 15%。
- 辐射爆炸覆盖以宝箱为中心的 3×3 格，范围内虫子死亡，并留下与感染尸体相同规则的污染。

### 3.4 第二纪元、供电与城市

- 中心反应堆是随机但可复现的 2×2 建筑：只在合法空地生成，不覆盖虫、物品、旧建筑、出生区或地图边界；位置进入存档。
- 中心反应堆建立后，第一纪元四类建筑隐藏血条并永久无敌；敌人不再以它们为攻击目标。
- 敌人遇到虫子会先攻击虫子，战略目标始终是中心反应堆。中心反应堆被摧毁时，所有虫子死亡并进入 Game Over。
- 第二纪元仅提供围墙、大楼、小型反应堆三种新建筑。
- 默认建造费用：围墙每格 50，大楼 1200，小型反应堆 800；全部进入统一平衡配置，不写死在逻辑中。
- 围墙和小型反应堆基础生命均为 60，不显示常驻血条，而用三段损坏外观表达状态。
- 敌人基础、二级、高级单位单次对墙伤害分别为 10、20、30，对应约 6、3、2 次攻击破墙。
- 小型反应堆占 1 格；被毁后自身及上下左右 4 格产生污染，并伤害/杀死范围内单位。
- 中心反应堆提供 100 单位电力；旧城区固定占用 80；每栋大楼消耗 20；每座小型反应堆增加 20。
- 需求不超过供给时为绿色，接近上限为黄色，超载为红色。超载时所有大楼产出按“供给÷需求”同比降低，不直接停摆。
- 大楼占 2×2 格，容量 10。空闲成虫会自动入住；入住后不再繁殖、不参加工作与战斗，并继续衰老、自然死亡。
- 大楼是安全建筑，敌人不把它作为目标且不能直接伤害居民；居民只会自然死亡，或在中心反应堆被摧毁时一同死亡。
- 鼠标悬停大楼可查看入住数、姓名、年龄、状态和个人贡献；死亡居民立即移出名单并释放床位。
- 每 10 秒结算一次：每位成年居民基础贡献 4 金币和 1 城市发展值；个体效率沿用虫子的差异参数，老年系数为 0.6；最后乘以供电比例。
- 城市发展目标为 1000。满员单楼约需 17 分钟，多楼配合小型反应堆应将正常第二纪元压缩到约 8–12 分钟。

### 3.5 第三纪元、进攻与反转

- 达到 1000 城市发展值后进入第三纪元；地图向右扩展一倍，载入与玩家区域等面积的敌方基地。
- 第三纪元开始后停止常规敌袭。已有士兵的剩余寿命乘 2，新生产士兵的总寿命乘 2。
- 士兵自动前往边界附近的固定集结点；每次新士兵加入显示“士兵正在集结”。
- 建筑面板在第三纪元只保留“进攻”按钮。点击后只派出当时已到达集结点的士兵；途中和之后出生的士兵留给下一波。
- 敌方基地默认有 6 座必须摧毁的核心建筑、外围墙和固定守军。建筑不可移动。
- 每次进攻时，敌方从生产建筑派出 3–5 个单位迎击；数量可小幅随机，但随机状态写入存档。
- 敌方建筑脱离伤害 8 秒后，以每秒最大生命 1% 的速度恢复。
- 数值目标：第一轮 10–14 名士兵可以突破外围并造成显著损伤；第二轮 14–18 名士兵通常可以获胜。最终以实际试玩调整，不强制脚本胜负。
- 胜利条件只计算 6 座核心建筑全部被毁；残余墙和普通机器人不阻塞结局。
- 胜利后锁定操作，显示大型“游戏胜利”，再黑屏切入独立结尾场景。
- 结尾右侧城市采用规整、近似玩家建筑语言但不复制玩家布局的镜像城市。虫子从建筑中出来并登上机器人，明确解释敌人是虫族驾驶的载具。
- “新时代，来临。”之后再次黑屏。德尔塔用受赠人姓名称呼新的博士，完成对白、眨眼、Logo 和制作人员字幕。

### 3.6 演出、输入、音频与稳定性

- 大演出由统一序列控制器接管暂停、输入权限、镜头、UI 和保存，避免对白结束后残留锁定。
- 第三纪元边缘滚屏同时提供 WASD/方向键替代操作；镜头不能越过双倍地图边界。
- 所有明确按钮共享 Normal/Hover/Pressed/Disabled 四态和轻微 UI 音；拖拽卡片不使用按钮态。
- 音乐切换采用淡入淡出；各纪元曲可无缝循环，统一使用一个主题动机，结尾将其变形回收。
- UI 音效限制同类短时间重复；战斗普通击打无声，大型世界事件可发声。
- 大规模第三纪元使用对象池管理士兵、敌人、粒子和浮字；目标为基准 1080p 下稳定 60 FPS，最低不低于 30 FPS。
- 每个执行日结束必须：运行可用的编译/自动测试、做对应场景冒烟检查、更新本文件进度、提交一次独立 Git commit，并推送当前分支。

## 4. 每日额度规则

- 以 Pro 月度可用量的比例控制，不假设一个公开固定 Token 数。
- 普通代码日目标不超过月额度 4%；美术、复杂演出和大规模整合日最多 5%。
- 阶段验收日优先控制在 3%，把额度留给后续不可预见问题。
- 没有执行任务的自然日消耗为 0%，不影响当前执行日编号；周六、周日也可以正常执行下一天。
- 禁用 Fast 模式，不开子代理；每次只处理当天范围。
- 若某日触及 5% 仍未通过验收，当天停止扩范围，把未完成项顺延并在进度表标红，不透支下一日额度。

## 7. 阶段性 Definition of Done

- 功能完整：三个纪元和最终反转都能由正常游玩触发，不依赖开发快捷键。
- 叙事正确：玩家姓名与受赠人姓名在各自位置使用正确；德尔塔台词顺序和语气忠实于叙事文档。
- 存档可靠：任意纪元保存退出后可继续；不存在复制资产、丢虫、重复纪元转场或重复寿命翻倍。
- 美术统一：新增角色、敌人、建筑、特效遵守现有像素比例、色板、过滤和层级规范。
- 交互清楚：明确按钮有 Hover/Pressed/Disabled；拖拽对象只有拖放反馈；所有关键操作都有可见结果。
- 音频克制：五组音乐完成情绪分工；UI 与大事件有声音；普通虫战不形成噪声墙。
- 性能可接受：第三纪元两侧实体达到设计规模时基准目标 60 FPS，最低不低于 30 FPS，无持续内存增长。
- 发布可复现：指定 Unity 版本可生成 Windows x64 构建，README 记载启动、存档位置、操作和已知限制。
