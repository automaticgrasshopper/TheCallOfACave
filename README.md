# 洞穴的呼唤 / The Call of the Cave

一个 2D 科幻洞穴虫群经营原型。玩家在荒芜洞穴中维持虫群的繁殖、劳作、医疗与防御；左侧营地逐步成为虫群城市，右侧是机械入侵者的战线。

完整规则、数值与分阶段任务见 [`GAMEPLAY_DESIGN.md`](GAMEPLAY_DESIGN.md)。

## 当前可玩循环

1. 开局拥有 300 金币、3 只幼虫和 2 枚虫卵；幼虫 120 金币，食物 30 金币。虫卵可任意拖动并在松手处继续孵化，只有拖到金币区才会卖出 100 金币。
2. 每只虫子出生时随机确定 160–180 秒（32–36 岁）的寿命、20–40 秒的破壳时间和 90%–110% 的生产效率，之后终生不变；低饱腹、低健康与感染会加速衰老。
3. 自由成虫约每 60 秒产卵；派去工厂、兵营、医院或研究院后停止繁殖。
   出生区只是生成点，所有未派驻虫子都会在全图不受控制地乱跑；敌人出现后，主动安置虫群本身就是玩法。
4. 工厂产物直接进入背包：一级初级零件、二级中级零件、三级随机高级零件 A／B。A+B 合成装备，装备+重型敌人特殊零件合成高级装备。
5. 老虫与阵亡士兵留下污染。90 秒后右侧出现机器人，之后每 45 秒一波；敌人就近攻击虫子或设施，设施受创会降级直至损毁，但敌人死亡不会产生污染。
6. 最后一只虫死亡时进入 **虫群失联 / Colony Lost**。

## UI 与状态反馈

- **HUD**：显示金币、食物、幼体／成体数量；`TIME`／`YEAR` 作为独立双行面板固定在 1920×1080 画面的顶部正中，不再占用资源卡。`TIME` 记录现实中的本局坚持时长；`YEAR` 从 `0001` 开始、每 5 秒增加一年，超过四位后显示五位并封顶 `99999`。暂停时两者都停止，虫群失联界面会保留最终值。
- **背包**：独立右侧栏中的 3×N 滚动 Grid，画面固定显示 3×3；空格不显示占位物，食物或工厂产物真正入库后才出现图标与数量。
- **指挥终端**：建造、研究、装备、背包和经济 HUD 收拢在独立右侧栏。装备页提供两级合成，每级都是两个半透明材料格和一个合成按钮，并实时显示库存。
- **虫体信息框**：普通虫显示永久代号、年龄、饱腹、工种和个体生产效率；士兵额外显示攻击、防御、当前／最大生命。年龄从 1 岁开始、每现实 5 秒增加 1 岁；饱腹使用 0–100% 正向数值。
- **Toast**：顶部非阻塞通知，用于繁殖、建造、训练、感染、入侵和资源反馈。
- **Alert**：居中的确认／取消弹窗，当前用于退出确认，可复用于危险操作。
- **Block**：全屏黑幕，场景或关卡切换时渐入渐出。
- **Game Over**：虫群全灭后的双语失败界面。

## 视觉与场景

视觉关键词是 **深层洞穴、赭金荒地、青绿色生物荧光、废土科技营地**。当前为 16-bit 像素科幻风格。流沙灾害已从当前版本移除；出生区保留为开放的孵化沙坑，它不是围墙建筑。

荒地已改为 Unity `Grid + Tilemap + RuleTile`：0.5 世界单位一格，四种沙地 Tile 自动随机变化，洞穴边缘在独立 Tilemap 层。开放育巢、隐藏建造网格、指挥侧栏、进度条、背包和全部 UI 都是 `Main.unity` 中可选中、移动、缩放和换图的真实对象。工厂、兵营、医院和研究院都以各自中空围墙 Prefab 由玩家拖拽放置；虫子、虫卵、敌人、博士与污染源也通过各自 Prefab 实例化到 `WorldRoot`。

标题界面使用全屏洞穴虫群插画与固定英文艺术 Logo。英文界面不重复显示文本标题；其他语言的本地化译名仅显示在 Logo 下方的副标题位。

全场景使用 640×360 参考分辨率的 Pixel Perfect Camera（Full HD 下整数 3 倍显示），所有世界与 UI 贴图统一 Point 采样、关闭 mipmap 和压缩；角色保留原有形态，同时保住砖缝、裂纹、铆钉与肢体结构等高密度像素细节。完整规则见 [`PIXEL_ART_DIRECTION.md`](PIXEL_ART_DIRECTION.md)。

背包图标位于 `Assets/Resources/Art/Inventory`，重型侵略者位于 `Assets/Resources/Art/Enemies`。工厂职员使用独立的 `bug_worker` 形象，普通成虫继续使用成虫行走表。

## 设施操作

- 从右侧建造页拖出工厂、兵营、医院或研究院，放到绿色的 0.5 单位网格上完成建造；红色表示越界、碰到出生坑或与其他设施的满级占地重叠。
- 四种设施按工厂 → 兵营 → 医院 → 研究院依次解锁，尚未满足前置条件的卡片会显示锁定。
- 每栋可放置设施从一级开始就预留 4.4×4.4 的满级占地；实际中空围墙会随等级按 1.15／1.50／1.90 半径扩张，因此后续升级不会挤到邻居。二级额外显示功能模块，三级再增加高级指挥节点和金色构件，并非只改变尺寸。
- 设施上方显示紧凑状态行（如“工厂 · Lv.1 · 1/3”，数字为现容纳／容量）；鼠悬停卡显示设施类型、等级、容纳和耐久，工厂卡额外显示当前生成物与出售价格。设施下方始终显示耐久血条。
- 按住 Shift 点击已建设施进行升级。
- 将自由成虫拖入工厂、兵营或研究院；将感染虫或受伤士兵拖入医院。
- 派驻虫子会在设施中心的小范围内徘徊，不会贴着围墙站立。军营训练条满后虫子转为士兵，但仍留在营内占用容量，需玩家手动拖出。工厂工人与士兵进入老年后自动退休、恢复自由身份并离开设施／岗位。
- 研究院普通点击开始医学博士培养。每名博士固定消耗 150 金币和 3 食物；其他三类研究人才位已预留。
- 购买食物后物品进入背包；食物可拖到场景任意位置成为地面实体。拖进设施时仅设施内成员争抢；拖在室外时感知范围内的自由虫会赶来，先碰到者吃掉并恢复饥饿与健康。
- 初级、中级零件可点击背包格出售。装备只通过拖到金币区出售；拖给士兵则装备，拖到地面会变成无功能的地面物件，点击可捡回背包。
- 普通敌人死亡掉落初级零件，重型敌人掉落特殊零件，均需点击地面掉落物拾取。高级装备会使士兵恢复到刚成年、满饱腹、满血并清除感染，攻击为普通士兵的 2.5 倍。

## 战斗与设施损毁

- 敌人持续选择最近的有效目标：最近的是虫子就攻击虫子，最近的是建筑就攻击建筑。
- 士兵和成年虫都能自动迎敌；普通成年虫伤害只有士兵的五分之一，单独迎战通常会牺牲。幼体与老虫被敌人命中即死亡。
- 一／二／三级设施最大耐久分别为 100／120／300。普通敌人每次攻击随机造成 5–15 伤害，高级敌人造成 10‑30 伤害；没有敌人再以该设施为目标时，每秒恢复最大耐久的 1%。高等级设施耐久耗尽时降低一级；一级设施耐久耗尽后损毁，内部所有成年成员立即恢复自由身份。
- 敌人死亡不产生污染；污染仍来自老虫死亡与士兵阵亡。

## 当前单局长度

玩法没有硬性胜利倒计时。第一只重型敌人约在 5 分 15 秒进入；当前一轮完整建设、防御与三级工厂体验约为 **6–10 分钟**。熟练玩家可以通过繁殖继续生存，但 **10–15 分钟后**敌人和事件组合会开始重复。后续优先补高级士兵动画、第二种普通敌人、重型敌人动画、建筑三级外观与 10 分钟事件／Boss，可把有效内容延长到约 20 分钟。

## 多语言约定（必须遵守）

项目以 `Assets/_Game/Data/LocalizationTable.asset` 为唯一文案表。任何面向玩家的新文字都必须：

1. 在 `LocalizationTable.cs` 的 `Keys` 中声明键名；
2. 在表中同时填写 `zh` 与 `en`；
3. UI 使用 `LocalizedText` 或 `LocalizationManager.Get(key)`；
4. 不在玩法脚本、预制体或场景中硬编码玩家可见文字。

## 运行

使用 Unity **2022.3.44f1** 或兼容的 2022 LTS 打开 `Assets/_Game/Scenes/Main.unity`。首次导入会自动烘焙场景设施、HUD 扩展、虫体状态条和动态实体 Prefab。若关闭了 Auto Refresh，请按一次 `⌘R`。

项目早期动画工作流参考 [`gorest-2d-animation-spritesheet-generator`](https://github.com/NO6KIKO/gorest-2d-animation-spritesheet-generator)；它不参与游戏运行时。

## 基线测试

保持 Unity Hub 已登录并运行，然后在仓库根目录运行 `Tools/run-baseline-smoke.sh`。入口会自动连接 Hub 的许可证服务，在临时副本中编译开发构建，并执行 360 秒主循环冒烟测试；正常游戏启动不会加载测试探针。环境、验收项和当前已知问题见 [`BASELINE_VALIDATION.md`](BASELINE_VALIDATION.md)。

## 档案领域模型测试

保持 Unity Hub 已登录并运行，然后在仓库根目录运行 `Tools/run-profile-domain-tests.sh`。入口会在临时项目副本中编译代码，并验证档案 ID、双姓名、累计时长、纪元、教程状态、结构版本和时间戳经过 JSON 序列化往返后完整保留。

## 档案仓库测试

运行 `Tools/run-profile-repository-tests.sh` 可验证 `Application.persistentDataPath/profiles` 仓库契约、档案索引、同目录临时文件原子替换、上一有效版本备份和损坏回退。测试只使用系统临时目录，并在结束后自动清理。

## 世界快照契约测试

运行 `Tools/run-world-snapshot-tests.sh` 可验证资金、年份、时长、随机种子，以及虫、蛋、库存、世界物品、设施、占用关系和计时器 DTO 经过 JSON 序列化往返后保持一致。

## 关于 AI 助手的美术产出边界

不具备内置图像生成工具的代码助手无法凭空画出角色、设施、背景、图标一类插画。它能做的包括程序化几何图（Tiles、网格、纯色/图案），以及原图之后的整条装配链——降采样／点采样量化、切帧、Unity import 设置、接线到 Prefab。

若要生成插画级 `*_raw.png`，需外部图像模型出图：gorest 走 **`GEMINI_API_KEY`**（Google Gemini，你自己的额度）或托管版 [sprite.gorest.ai](https://sprite.gorest.ai/)（登录免 key）。提供 Gemini key 后，助手可脚本调用其出图接口，从生成到入库全包。

## 外部 AI 助手协作边界（必须遵守）

任何外部 AI 助手只承担与主工程完全解耦、可单独审查和撤销的小任务。本项目的架构、主流程和最终集成统一由 Codex 负责。

允许范围：

1. 拉取最新代码，在独立分支或 Worktree 中工作；
2. 新增自包含的 Shader、材质、后处理或小型屏幕表现组件；
3. 新增不依赖主流程、通过明确公开接口接入的独立小功能；
4. 处理已提供原图后的切片、量化、导入设置等机械美术装配；
5. 按明确任务处理原创或已确认授权的音乐、音效素材及其独立导入配置；
6. 提交单一目的 commit，并提供提交号、修改文件、测试结果和接入说明。

禁止范围：

1. 不修改 `Assets/_Game/Scripts/Core`、`Managers`、`Persistence` 中的架构和数据契约；
2. 不修改存档格式、稳定 ID、枚举数值、主循环、经济数值、纪元规则、教程或叙事流程；
3. 不修改 `Main.unity`、现有共享 Prefab、`ProjectSettings`、`Packages`、场景生成/烘焙脚本；
4. 不修改 `GAME_FLOW_NARRATIVE.md`、`IMPLEMENTATION_SCHEDULE.md`、`BASELINE_VALIDATION.md` 或完成进度；
5. 不移动、重命名、覆盖或删除现有资产及其 `.meta` 文件；
6. 不直接提交、合并或推送到 `main`，不自行接线到共享场景；
7. 不同时打开或修改 Codex 正在使用的同一 Unity 工程目录。

外部协作交付默认停在独立分支，由 Codex 复核代码和许可证来源、运行 Unity 编译与回归测试，并完成最终接线和合并。未经明确列入允许范围的文件，一律视为不可修改。
